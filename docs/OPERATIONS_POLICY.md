# CartPick 운영 정책 v0.1 (MVP)

## 1. 문서 목적

이 문서는 CartPick 서비스의 **상품/리뷰/홈 운영 기준**을 정의한다.

**목표:**
- 데이터 일관성 유지
- 운영 판단 기준 통일
- 서비스 품질 유지
- 향후 확장 시 기준점 제공

---

## 2. 서비스 핵심 원칙

1. **리뷰는 상품(Product) 기준으로 통합한다.**
2. **리테일러는 판매처/구매처 정보로만 사용한다.**
3. **홈은 자동 정렬이 아닌 발견(Discovery) 중심 운영을 지향한다.**
4. **초기에는 운영(수동) 중심으로 품질을 확보한다.**

---

## 3. 리뷰 정책

### 3.1 리뷰 작성 정책

- 유저는 **상품당 1개**의 리뷰만 작성 가능
- 기존 리뷰는 수정 가능
- 리뷰 본문 최소 길이는 **없음**
- **별점만으로도** 리뷰 작성 가능
- 구매처(retailer_id)는 **선택 사항**

### 3.2 리뷰 공개 정책

| 유저 상태 | 열람 범위 |
|-----------|-----------|
| 비로그인 | 대표 리뷰 1개만 열람 가능 |
| 로그인 | 전체 리뷰 열람 가능 |

> 비로그인 제한은 회원가입 전환을 위한 장치. Post-MVP에서 완화 가능.

### 3.3 리뷰 내용 정책

**허용:**
- 개인 경험 기반 평가
- 맛, 가성비, 양 등 솔직한 의견
- 호불호 표현

**제한:**
- 욕설, 비방, 특정인 비하
- 광고/홍보성 글
- 의미 없는 반복 글
- 링크 포함 글 (광고 의심)
- 개인정보 포함 글

### 3.4 리뷰 모더레이션 상태

관리자는 리뷰를 다음 상태로 관리한다:

| 상태 | 설명 | 유저 노출 |
|------|------|-----------|
| `approved` | 정상 노출 (기본값) | ✅ |
| `pending` | 검토 대기 | ❌ |
| `hidden` | 임시 숨김 | ❌ |
| `rejected` | 영구 차단 | ❌ |

### 3.5 신고 처리 기준

- 신고 누적 3건 이상 → 자동 `pending` 전환 (관리자 검토 필요)
- 동일 유저 여러 번 신고 → 1회로 집계
- 관리자 판단:
  - 정상: `approved` 복귀
  - 경미: `hidden` 처리
  - 심각: `rejected` 처리

## 4. 상품 정책

### 4.1 상품 정의

> 상품(Product)은 **리뷰가 귀속되는 단위**이다.

### 4.2 상품 생성 기준

다음 조건이 만족되면 **별도의 상품**으로 등록한다:

- 브랜드가 다른 경우
- 상품명이 다른 경우
- 용량/사이즈가 다른 경우
- 맛/성분/구성 차이가 있는 경우 (예: 민트초코 vs 일반 초코)

### 4.3 상품 병합 정책

동일 상품이 중복 등록된 경우, **관리자에 의해 수동 병합**한다.

**병합 기준:**
- 동일 브랜드
- 동일 또는 유사 상품명
- 동일 용량/구성
- (선택) 동일 바코드

**병합 시 처리:**
1. 리뷰 → 기준 상품으로 이동
2. 리테일러 연결(`retailer_products`) → 기준 상품으로 이관
3. 즐겨찾기(`bookmarks`) → 기준 상품으로 이관
4. 기존 상품 → `status = 'inactive'` 처리 + `product_aliases` 생성
5. `product_merge_histories`에 이력 기록

### 4.4 상품 리뉴얼 정책

**MVP 기준:**
- 상품 리뉴얼 여부는 **자동 판단하지 않음**
- 리뉴얼로 인한 새로운 리뷰 허용 여부는 Post-MVP 정책으로 정의

### 4.5 상품 상태

| 상태 | 설명 |
|------|------|
| `draft` | 작성 중 (미노출) |
| `active` | 정상 노출 |
| `inactive` | 비활성 (검색 X, 과거 리뷰는 유지) |

## 5. 리테일러 및 가격 정책

### 5.1 리테일러 역할

- 상품의 **판매처 정보** 제공
- 리뷰 필터 기준으로 사용
- **리뷰는 리테일러가 아닌 상품에 귀속됨**

### 5.2 리테일러 런칭 상태

| 상태 | 설명 |
|------|------|
| `active` | 정상 운영 (GS25, CU, 세븐일레븐, 이마트24) |
| `coming_soon` | 출시 예정 (Costco, 트레이더스) |
| `inactive` | 비활성 |

### 5.3 가격 정책

가격은 **참고값(reference price)** 으로 정의한다.

**표시 방식:**
- 가격: `2,500원`
- 보조 정보: "최근 확인 가격"
- 확인 시점 표시 (`last_synced_at`)

### 5.4 가격 해석 기준

- 가격은 **실제와 다를 수 있음**
- 사용자는 참고용으로만 활용
- **정확성 보장은 하지 않음** (이용약관 명시 필요)
- 가격 정보가 없는 경우 "가격 정보 없음" 노출

### 5.5 할인 정보

- `sale_price`가 `price`보다 낮으면 할인 표시
- 할인율 표시 가능
- 할인가는 시점성이 강하므로 `last_synced_at`와 함께 노출

## 6. 홈 운영 정책

### 6.1 운영 원칙

- 홈은 **발견 중심(Discovery) UX**를 제공한다
- 단순 정렬이 아닌 **큐레이션 기반** 구성
- 초기에는 **수동 중심**으로 운영

### 6.2 운영 비율 (MVP)

| 유형 | 비율 | 설명 |
|------|------|------|
| 수동 운영 | **70%** | 운영자가 선택한 섹션/상품 |
| 자동 운영 | **30%** | 쿼리 조건 기반 자동 노출 |

### 6.3 홈 섹션 유형

**수동 운영 섹션 (`is_manual = true`)**
- 이번 주 신상 (운영자 선택 가능)
- 요즘 핫한 젤리 (카테고리 큐레이션)
- 기획/추천 상품
- 시즌 이벤트 (예: 여름 아이스크림 특집)

**자동 운영 섹션 (`is_manual = false`)**
- 지금 리뷰 급증 상품 (최근 24h 리뷰 증가량)
- 리테일러별 인기 상품 (GS25, CU 각각)
- 지금 할인 중 (할인 적용 상품)

### 6.4 운영 권한

**모든 홈 섹션은 관리자가 수정 가능해야 한다.**

이유:
- 초기 데이터 부족 보완
- 향후 광고/프로모션 대응
- 자동 쿼리 결과가 부실할 때 개입 필요

### 6.5 MVP 초기 섹션 구성

| 순서 | 섹션 | 타입 | 모드 |
|------|------|------|------|
| 1 | 🔥 이번 주 편의점 신상 | `new_this_week` | 자동 (14일 내, is_new) |
| 2 | 📈 지금 핫한 상품 | `trending` | 자동 (24h 리뷰 증가) |
| 3 | 🏪 GS25 인기 상품 | `popular_by_retailer` | 자동 |
| 4 | 🏪 CU 인기 상품 | `popular_by_retailer` | 자동 |
| 5 | 🎯 카테고리 | `categories` | 수동 |
| 6 | 💰 지금 할인 중 | `on_sale` | 자동 (데이터 부족 시 숨김) |

### 6.6 섹션 숨김 기준

- 자동 섹션은 상품 **3개 미만** 시 자동 숨김
- 수동 섹션은 운영자가 명시적으로 `is_active = false` 설정

## 7. 신상품 정책

### 7.1 신상품 정의

상품 등록 후 **일정 기간** 동안 신상품으로 간주한다.

### 7.2 신상품 기준

| 용도 | 기간 | 근거 |
|------|------|------|
| **NEW 뱃지** | 등록 후 **14일** | 상품 카드 `🆕` 표시 |
| **"이번 주 신상" 섹션** | 최근 **7일** | 홈 최상단 섹션 |
| **얼리 어답터 뱃지** | 등록 후 3일 내 리뷰 | 유저 보상 |

### 7.3 UI 반영

- 상품 카드: `🆕 NEW` 뱃지 (14일)
- 홈: "이번 주 편의점 신상" 섹션 (7일)
- 리뷰: "얼리 어답터" 유저 뱃지 표시

## 8. 관리자 운영 정책

### 8.1 관리자 역할

관리자는 다음 업무를 수행한다:

1. 상품 등록 및 수정
2. 리테일러 연결 (retailer_products)
3. 상품 병합
4. 리뷰 모더레이션 (정상/숨김/차단)
5. 홈 섹션 큐레이션
6. 신고 처리

### 8.2 운영 우선순위 (일일 업무)

| 우선순위 | 업무 | 빈도 |
|----------|------|------|
| 1 | 신고 리뷰 처리 | 매일 |
| 2 | 신상품 등록 | 매일 |
| 3 | 리뷰 없는 상품 보완 (시드 리뷰) | 매일 |
| 4 | 홈 섹션 관리 | 주 2~3회 |
| 5 | 상품 중복 정리 | 주 1회 |

### 8.3 관리자 권한 레벨 (Post-MVP)

| 역할 | 권한 |
|------|------|
| `owner` | 전체 권한 (관리자 추가/삭제 포함) |
| `admin` | 상품/리뷰/홈 CRUD |
| `operator` | 상품 등록, 리뷰 모더레이션만 |

MVP에서는 단일 `admin` 계정 운영.

### 8.4 운영 로그

모든 관리자 액션은 `admin_logs`에 기록:
- 누가 (`admin_id`)
- 언제 (`created_at`)
- 무엇을 (`action_type`, `target_type`, `target_id`)
- 어떻게 (`payload_json`)

## 9. 데이터 품질 정책

### 9.1 필수 데이터 기준

**상품 등록 시 필수:**
- 브랜드
- 카테고리
- 상품명
- 대표 이미지

**상품 등록 시 권장:**
- 정규화명 (`normalized_name`)
- 한 줄 요약 (`summary_line`)
- 바코드
- 용량

### 9.2 중복 방지

- `normalized_name` 기반 중복 후보 자동 탐지 (Post-MVP)
- 등록 시 유사 상품 경고 (Post-MVP)
- 바코드 중복 시 경고

### 9.3 데이터 정제

- 주기적 `normalized_name` 재계산
- Unused 브랜드/카테고리 정리
- 비활성 리테일러 연결 정리

## 10. KPI 관련 정책

### 10.1 초기 KPI (MVP)

1. **상품당 평균 리뷰 수** — 목표 3개 이상
2. **리뷰 작성 전환율** — 목표 5%
3. **검색 → 리뷰 작성 비율** — 목표 10%

### 10.2 운영 개입 기준

| 상황 | 개입 |
|------|------|
| 상품 등록 후 3일 리뷰 0개 | Seed 리뷰 작성 |
| 신고 리뷰 5건 이상 | 즉시 검토 |
| 홈 섹션 클릭률 저조 | 섹션 재편집 |

---

## 11. 확장 고려 항목 (Post-MVP)

다음 항목은 MVP 이후 확장 대상:

- 상품 자동 매칭 (이름/바코드 유사도 기반)
- 병합 후보 자동 추천
- 가격 자동 수집 (크롤링 또는 API 연동)
- 홈 자동 추천 알고리즘 강화 (개인화)
- 리뷰 요약 AI (LLM 기반 한 줄 요약)
- 이미지 리뷰
- 바코드 스캔
- 유저 뱃지/레벨 확장
- 글로벌 확장 (다국가)
- 관리자 역할 세분화

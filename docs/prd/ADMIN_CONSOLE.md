# CartPick 관리자 콘솔 설계

## 0. 관리자 페이지의 목표

관리자 페이지는 **운영 콘솔(Operations Console)** 이다. 예쁠 필요 없다. 빠르고 명확해야 한다.

관리자가 해결해야 할 일은 4가지:

1. **상품을 등록/수정**한다
2. **상품을 리테일러에 연결**한다
3. **리뷰를 관리**한다
4. **홈에 노출할 데이터를 운영**한다

운영자가 매일 들어와서 해야 하는 일이 **바로 보여야 한다**.

---

## 1. 운영 핵심 4가지 (잊지 말 것)

| 운영 활동 | 화면 | 빈도 |
|-----------|------|------|
| 신상품을 빨리 넣는다 | 상품 등록 → 리테일러 연결 | 매일 |
| 중복 상품을 정리한다 | 상품 병합 | 주간 |
| 이상 리뷰를 처리한다 | 리뷰 관리 | 매일 |
| 홈을 계속 살아있게 한다 | 홈 운영 | 주간 |

🏪 리테일러 관리

### MVP 핵심 4개 화면

구분선 위 4개 + 홈 운영 = **MVP 핵심**

나머지는 2차/3차 우선순위.

- 최근 등록된 상품 10개

- 신고된 리뷰 10개
- 리뷰 없는 신상품 10개

## 6. 화면 3: 상품 등록/수정

폼 기반 화면. 섹션 분할.

### 섹션 A. 기본 정보
- 상품명 *필수*
- 정규화 상품명 (자동 생성 + 수동 수정 가능)
- 브랜드 선택 (**자동완성 드롭다운 필수**)
- 카테고리 선택 (**자동완성 드롭다운 필수**)
- 국가 (기본: 한국)
- 설명 (textarea)
- 한 줄 요약 (`summary_line`) ← 상품 상세에 노출
- 대표 이미지 업로드

### 섹션 B. 식별 정보
- 바코드
- 용량 값 (숫자)
- 용량 단위 (g / ml / 개 등)
- 패키지 타입

### 섹션 C. 운영 정보
- 상태 (active / inactive / draft)
- 추천 노출 여부
- 관리자 메모

### 하단 버튼
- `저장`
- `임시 저장` (draft)
- `저장 후 리테일러 연결하기` ← 가장 많이 쓰는 동선

### ⚠️ UX 필수 조건

> **브랜드와 카테고리는 자동완성 드롭다운이 필수다.**
>
> 매번 새로 입력하게 하면 데이터가 금방 더러워진다.

## 7. 화면 4: 리테일러 상품 관리 (핵심)

> CartPick의 핵심 차별점. 상품만 있으면 안 되고 **어느 마트에서 파는지**가 있어야 한다.

### 상단
```
리테일러 상품 관리                    [+ 리테일러 상품 추가]
```

### 필터
- 리테일러
- 카테고리
- 판매 여부
- 신상품 여부
- 가격 입력 여부
- 마지막 동기화 시점

### 목록 테이블 컬럼
- ID
- 리테일러
- 상품명 (내부)
- 리테일러 노출명
- 가격
- 할인가
- 판매 여부
- 신상품 여부
- 한정 여부
- 마지막 수정일
- 액션

### 등록/수정 폼
- 리테일러 선택
- **연결할 상품 선택 (⚠️ 자동완성 검색 필수)**
- 리테일러 내 상품명
- 상품 URL
- 가격
- 할인가
- 통화 (기본 KRW)
- 판매 여부
- 신상품 여부
- 한정 여부
- source_type (manual / crawled / user_submitted)
- last_synced_at

### ⚠️ 치명적 UX 포인트

> **상품 검색은 반드시 쉬워야 한다.**
>
> 상품 연결할 때 자동완성 검색이 없으면 운영 피로도가 치솟는다. 이게 없으면 관리자가 신상품 추가를 포기한다.

## 9. 화면 6: 홈 운영

> 홈은 자동 정렬이 아니라 **편집 홈**이다. 이 화면이 있어야 홈을 살아있게 유지할 수 있다.

### 관리 대상 섹션
- 이번 주 신상
- 지금 핫한 상품
- GS25 인기 상품
- CU 인기 상품
- 지금 할인 중
- 요즘 핫한 젤리 (카테고리 트렌드)

### 화면 구조

섹션 리스트가 세로로 보임:

```
┌─────────────────────────────────────────────────────┐
│ 🔥 이번 주 신상                          [수정]      │
│    활성: ✅  순서: 1  상품: 12개                     │
├─────────────────────────────────────────────────────┤
│ 📈 지금 핫한 상품                         [수정]      │
│    활성: ✅  순서: 2  상품: 10개                     │
├─────────────────────────────────────────────────────┤
│ 🏪 GS25 인기 상품                         [수정]      │
│    활성: ✅  순서: 3  상품: 10개                     │
├─────────────────────────────────────────────────────┤
│ 🏪 CU 인기 상품                           [수정]      │
│    활성: ✅  순서: 4  상품: 10개                     │
├─────────────────────────────────────────────────────┤
│ 💰 지금 할인 중                           [수정]      │
│    활성: ❌  순서: 5  상품: 0개                      │
└─────────────────────────────────────────────────────┘

[+ 섹션 추가]
```

### 섹션 수정 화면

- 섹션 제목
- 섹션 타입 (`new_this_week`, `trending`, `popular_by_retailer`, `category_trending`, `on_sale`, `manual` 등)
- 자동/수동 여부
- 자동 조건 (타입별)
- 수동일 경우 상품 직접 선택
- 정렬 기준
- 노출 개수
- 시작일 / 종료일 (캠페인성 섹션)
- 활성 여부

### 섹션 타입별 예시

**이번 주 신상 (자동)**
```
타입: new_this_week
자동 조건: is_new=true, created_at >= NOW() - 14일
정렬: created_at DESC
노출 수: 12
```

**GS25 인기 상품 (자동)**
```
타입: popular_by_retailer
리테일러: GS25
정렬: 최근 리뷰 수 DESC
노출 수: 10
```

**요즘 핫한 젤리 (자동)**
```
타입: category_trending
카테고리: 젤리
정렬: 최근 7일 리뷰 증가량 DESC
노출 수: 10
```

**운영자 선택 (수동)**
```
타입: manual
상품 직접 선택 (드래그 정렬)
노출 수: 제한 없음
```

## 15. 운영자 핵심 플로우

관리자가 실제로 가장 많이 하는 3가지 흐름:

### 🔥 플로우 1: 신상품 추가

```
대시보드
  ↓
[+ 상품 등록] 클릭
  ↓
기본 정보 입력 (브랜드/카테고리 자동완성)
  ↓
[저장 후 리테일러 연결하기]
  ↓
리테일러 선택 + 가격 입력
  ↓
[홈 운영] → "이번 주 신상" 섹션 확인
  ↓
(필요 시) 시드 리뷰 작성
```

### 🔥 플로우 2: 중복 상품 정리

```
상품 관리
  ↓
검색 또는 필터
  ↓
유사 상품 발견
  ↓
[상품 병합] 클릭
  ↓
기준 상품 선택 → 병합 실행
```

### 🔥 플로우 3: 이상 리뷰 처리

```
대시보드 "운영 필요 항목" → 신고 대기 클릭
  ↓
리뷰 상세 패널
  ↓
판단 → [숨김] 또는 [정상 노출]
  ↓
운영 메모 기록
```

> **이 3개 흐름이 부드러우면 어드민은 성공이다.**

## 18. 데이터 모델 영향 (ERD 변경)

관리자 기능에 필요한 추가 테이블:

### `home_sections` (MVP 필수 — 홈 운영)

```sql
CREATE TABLE home_sections (
    id SERIAL PRIMARY KEY,
    country_id INTEGER NOT NULL REFERENCES countries(id),
    title VARCHAR(100) NOT NULL,
    emoji VARCHAR(10),
    section_type VARCHAR(30) NOT NULL,
    -- 자동/수동 구분
    is_manual BOOLEAN NOT NULL DEFAULT false,
    -- 자동 조건 (JSONB)
    auto_config JSONB,
    -- 정렬/노출
    display_order INTEGER NOT NULL DEFAULT 0,
    display_count INTEGER NOT NULL DEFAULT 10,
    -- 활성 & 기간
    is_active BOOLEAN NOT NULL DEFAULT true,
    starts_at TIMESTAMPTZ,
    ends_at TIMESTAMPTZ,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

### `home_section_items` (수동 섹션용)

```sql
CREATE TABLE home_section_items (
    id SERIAL PRIMARY KEY,
    section_id INTEGER NOT NULL REFERENCES home_sections(id) ON DELETE CASCADE,
    product_id INTEGER NOT NULL REFERENCES products(id),
    display_order INTEGER NOT NULL DEFAULT 0,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    UNIQUE (section_id, product_id)
);
```

### `product_aliases` (상품 병합용)

```sql
CREATE TABLE product_aliases (
    id SERIAL PRIMARY KEY,
    product_id INTEGER NOT NULL REFERENCES products(id),
    alias_name VARCHAR(200) NOT NULL,
    source VARCHAR(50),
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

### `admins` (관리자 계정)

```sql
CREATE TABLE admins (
    id SERIAL PRIMARY KEY,
    email VARCHAR(255) NOT NULL UNIQUE,
    password_hash VARCHAR(255) NOT NULL,
    name VARCHAR(50) NOT NULL,
    role VARCHAR(30) NOT NULL DEFAULT 'operator',
    is_active BOOLEAN NOT NULL DEFAULT true,
    last_login_at TIMESTAMPTZ,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

### `admin_logs` (운영 이력)

```sql
CREATE TABLE admin_logs (
    id BIGSERIAL PRIMARY KEY,
    admin_id INTEGER NOT NULL REFERENCES admins(id),
    action VARCHAR(50) NOT NULL,
    target_type VARCHAR(30),
    target_id INTEGER,
    changes JSONB,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

### `reviews.admin_note` (리뷰 운영 메모)

```sql
ALTER TABLE reviews ADD COLUMN admin_note TEXT;
```

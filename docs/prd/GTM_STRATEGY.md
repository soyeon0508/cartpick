# CartPick Go-to-Market 전략

## 0. 전체 전략 한 줄 요약

> **"편의점 신상품 → 궁금 → 검색 → 리뷰 없음 → 내가 첫 리뷰 작성"**
>
> 이 루프를 만드는 것이 CartPick의 핵심 성장 엔진이다.

---

## 1. 유저 유입 전략 (Acquisition)

### 🎯 초기 타겟

명확한 1차 타겟:

> **"편의점 신상 먹어보는 사람"**

- 대학생
- 20~30대 직장인
- 편의점을 주 3회 이상 방문하는 사람
- SNS에서 신상 리뷰 콘텐츠를 소비하는 사람

### 📌 유입 채널 1: SNS (가장 중요)

**왜 SNS인가:**
- 이미 "신상 간식 리뷰" 콘텐츠 시장이 존재
- TikTok / 인스타 릴스 / 유튜브 쇼츠에서 검증된 포맷
- 타겟 유저가 이미 머무는 곳

**콘텐츠 유형:**

| 유형 | 포맷 예시 |
|------|-----------|
| **신상 리뷰** | "GS25 신상 젤리 먹어봤는데… 이건 좀 충격이다" |
| **비교 콘텐츠** | "CU vs GS25 젤리 뭐가 더 맛있냐면…" |
| **실패 방지** | "이거 사기 전에 꼭 보세요 (별점 2.1 이유 있음)" |
| **랭킹** | "이번 주 CartPick 평점 TOP 5" |

**CTA (매우 중요):**

영상 마지막에 반드시:
> "CartPick에서 리뷰 확인 가능"

### 📌 유입 채널 2: 검색 (SEO + 앱스토어)

**유저 실제 검색어:**
- "GS25 신상 젤리"
- "CU 라면 맛있음?"
- "코스트코 추천템"
- "편의점 신상 후기"

**전략:**

| 전략 | 시점 |
|------|------|
| 웹 상품 상세 페이지 (SEO용) | Post-MVP |
| 앱스토어 키워드 최적화 (ASO) | MVP 런칭 직후 |
| Google/Naver 검색 인덱싱 | Post-MVP |

### 📌 유입 채널 3: 커뮤니티

**타겟 커뮤니티:**
- 디시 편의점 갤러리
- 더쿠
- 클리앙
- 대학 커뮤니티 (에브리타임 등)
- 네이버 카페 (편의점 관련)

**원칙:**
- 링크 던지기 ❌
- 정보 제공 + 자연스러운 언급 ⭕

**예시:**
> "요즘 GS25 젤리 리뷰 모아놓은 앱 있는데, 이거 보고 사면 실패 줄어듬"

## 2. 리뷰 확보 전략 (Activation)

> ⚠️ **이게 Acquisition보다 더 중요하다.**
>
> 리뷰 없는 리뷰 앱은 죽는다.

### 🎯 핵심 원칙

> **"리뷰를 쓰게 하지 말고, 쓰고 싶게 만들어라"**

### 💡 전략 1: 첫 리뷰 작성 유도

**리뷰 0개인 상품 UX:**

```
┌──────────────────────────┐
│  🏆 첫 리뷰를 남겨주세요  │
│                          │
│  이 상품의 첫 리뷰어가    │
│  되면 특별 뱃지를 드려요  │
│                          │
│  [ 첫 리뷰 작성하기 ]    │
└──────────────────────────┘
```

**심리적 포인트:**
- 사람은 "첫 번째"를 좋아한다
- 뱃지/배지 같은 작은 보상으로 동기 부여
- "다른 사람에게 큰 도움이 됩니다" 메시지

### 💡 전략 2: 구매 직후 트리거

**상황 인식:**
- 편의점에서 방금 먹음
- 지금이 평가 의욕 최고점
- 집 가면 잊어버림

**해결:**

| 방법 | 시점 | 구현 |
|------|------|------|
| **방법 A** | Post-MVP | 바코드 스캔 → 바로 리뷰 작성 화면 |
| **방법 B** | MVP | 검색 → 상품 진입 → 리뷰 작성 (3탭 이내) |

**핵심 UX 목표:**
> **"상품 발견부터 리뷰 작성 시작까지 3초"**

### 💡 전략 3: Seed 리뷰 (MVP 필수)

> 🚨 **이게 없으면 서비스가 죽는다.**

**방법:**
- 팀이 직접 작성
- 상품 200~300개
- 각 상품당 1~3개 리뷰
- **총 300~500개 seed 리뷰 목표**

**주의 사항:**
- 너무 티나면 안 됨
- 실제 후기처럼 자연스럽게
- 극단적 평가(★1, ★5)보다 ★3~4 위주
- 구매처, 짧은 소감, 구체적 표현 필수

### 💡 전략 4: "나도 한마디" 경량 리뷰 UX

**리뷰 쓰기 부담 감소:**

```
이 상품 어땠나요?

⭐️⭐️⭐️⭐️☆

□ 맛있어요
□ 가성비 좋아요
□ 양이 적어요
□ 재구매 의사 있음

한 줄 평 (선택):
[_________________________]

   [ 리뷰 등록 ]
```

**설계 원칙:**
- 별점만 있어도 제출 가능
- 태그 기반 빠른 입력
- 긴 글은 선택 사항
- **필수 입력 최소화**

### 💡 전략 5: 비교 욕구 활용

사람은 비교할 때 리뷰를 쓰게 된다.

**UX 아이디어:**

```
CU vs GS25
어디 젤리가 더 맛있어요?

[ CU 리뷰 작성 ]  [ GS25 리뷰 작성 ]
```

### 💡 전략 6: 논쟁 유발 상품 우선 등록

**초기 상품 큐레이션 원칙:**

리뷰가 많이 달릴 만한 상품 우선:
- 민트초코 계열
- 매운맛 (불닭, 틈새라면 등)
- 호불호 젤리 (젤리빈, 사탕 계열)
- 신상/한정판
- 콜라보 상품

## 3. 리텐션 전략 (Retention)

### 💡 전략 1: 신상품 알림

> "GS25에 신상 5개가 나왔어요"

- 푸시 알림 (Post-MVP)
- 홈 화면 "신상품" 섹션 (MVP)

### 💡 전략 2: 관심 카테고리 추천

> "젤리 좋아하는 당신을 위한 추천"

- 유저의 리뷰/즐겨찾기 기반 추천

### 💡 전략 3: 내 리뷰 반응 알림

> "당신의 리뷰에 좋아요 5개가 달렸어요"

- 가장 강력한 재방문 트리거
- 리뷰 작성자에게 성취감 제공

## 4. MVP 실행 플랜

### 🔥 Week 0 (출시 전 준비)

- [ ] 상품 200~300개 등록 (관리자 수동)
- [ ] Seed 리뷰 300~500개 작성 (팀 주도)
- [ ] 논쟁 유발 상품 우선 배치
- [ ] SNS 콘텐츠 10개 사전 제작

### 🔥 Week 1 (출시 직후)

- [ ] SNS 콘텐츠 업로드 시작 (TikTok, 인스타 릴스)
- [ ] 지인 30명 초대 및 피드백 수집
- [ ] 커뮤니티 5곳에 자연스럽게 노출
- [ ] 첫 리뷰 작성 유도 UX 모니터링

### 🔥 Week 2~3 (분석 & 최적화)

- [ ] 어떤 상품에 리뷰가 많이 달리는지 분석
- [ ] 인기 카테고리에 리소스 집중
- [ ] 이탈 구간 분석 (검색 → 상품 → 리뷰 작성 깔때기)
- [ ] 논쟁 상품 추가 등록

## 5. 핵심 KPI

초기에는 다음 3가지 지표만 본다:

### 🎯 KPI 1: 상품당 평균 리뷰 수

- **목표:** 최소 3개 이상
- **왜:** 리뷰 1~2개 상품은 유저가 신뢰하지 않음

### 🎯 KPI 2: 리뷰 작성 전환율

- **정의:** 상품 상세를 본 유저 중 리뷰를 작성한 비율
- **목표 (MVP):** 5% 이상
- **왜:** 리뷰 확보 속도의 직접 지표

### 🎯 KPI 3: 검색 → 리뷰 작성 비율

- **정의:** 검색으로 상품에 도달한 유저 중 리뷰를 작성한 비율
- **목표 (MVP):** 10% 이상
- **왜:** "궁금해서 검색 → 리뷰 없음 → 내가 쓴다" 루프의 핵심 지표

### 보조 지표 (참고용)

| 지표 | 설명 |
|------|------|
| DAU / MAU | 일간/월간 활성 유저 |
| 리뷰당 좋아요 수 | 리뷰 품질 간접 지표 |
| 7일 리텐션 | 재방문률 |
| 리뷰 0개 상품 비율 | 낮을수록 좋음 |

| 기능 | 우선순위 | 목적 |
|------|----------|------|

## 7. 리스크 & 대응

| 리스크 | 대응 |
|--------|------|
| 리뷰가 안 쌓임 | Seed 리뷰 강화, 논쟁 상품 우선, 첫 리뷰 뱃지 |
| 유저가 검색해도 상품이 없음 | "상품 등록 요청" 기능, 인기 상품 우선 등록 |
| SNS 콘텐츠 ROI 저조 | 비교 콘텐츠 등 검증된 포맷 집중 |
| 가짜/광고 리뷰 유입 | 신고 기능, 모더레이션 대시보드, 초기에는 수동 검토 |
| 경쟁 서비스 등장 | "첫 리뷰어 선점" 효과 극대화, 커뮤니티 기반 방어 |

# CartPick 홈 화면 UX 설계

## 0. 홈의 목표

홈 화면은 딱 3가지 일을 한다:

1. **"지금 뭘 사야 할지" 알려준다** (Decision)
2. **"새로운 상품 발견"을 만든다** (Discovery)
3. **"리뷰 보러 들어가게 만든다"** (Engagement)

> CartPick은 리뷰 앱이지만, **홈은 탐색(Discovery) 중심**이어야 한다.

---

## 1. 설계 원칙

### ❌ 피해야 할 약한 구조

```
- 인기 상품
- 신상품
- 평점 높은 상품
```

**문제:**
- 너무 generic
- 클릭 이유 없음
- 앱 열자마자 흥미 없음

### ✅ 올바른 구조: "상황 + 시간 + 마트" 기반

홈의 모든 섹션은 3가지 축 중 하나 이상을 충족해야 한다:

| 축 | 질문 | 예시 |
|----|------|------|
| **"지금"** (시간성) | 지금 뭐가 핫해? | 이번 주 신상, 오늘 리뷰 급증 |
| **"어디서"** (마트) | 내가 가는 편의점에 뭐가 있어? | GS25 인기, CU 인기 |
| **"왜"** (상태) | 왜 이 상품을 봐야 해? | 리뷰 폭발 중, 할인 중, 첫 리뷰어 모집 |

## 3. 섹션별 상세 설계

### 🔥 Section 1. "이번 주 편의점 신상" (최우선)

> 홈 1순위. 앱 열자마자 첫 노출.

**왜 중요한가:**
- 편의점 유저 = 신상 중심 소비
- "궁금 → 클릭 → 리뷰 없음 → 내가 첫 리뷰" 루프의 시작점

**쿼리 조건:**
- `retailer_products.is_new = true`
- `retailer_products.created_at >= NOW() - INTERVAL '7 days'`
- 활성 편의점 리테일러만

**카드 구성:**
```
┌─────────────────┐
│                 │
│   [상품 이미지]  │
│                 │
│  🆕 NEW        │
├─────────────────┤
│ 하리보 피크닉   │
│ ⭐ - · 💬 0개   │
│ GS25            │
└─────────────────┘
```

**핵심 UX:** 리뷰 0개 상품에는 `💬 첫 리뷰 작성` 강조 표시 → 첫 리뷰어 뱃지 유도.

---

### 📈 Section 2. "지금 핫한 상품" (FOMO 유도)

> 최근 리뷰가 급증한 상품. 시간성 강조.

**쿼리 조건:**
- 최근 24시간 / 7일 리뷰 증가량 기준 정렬
- `SELECT product_id, COUNT(*) FROM reviews WHERE created_at >= NOW() - INTERVAL '24 hours' GROUP BY product_id ORDER BY COUNT(*) DESC`

**카드 뱃지:**
- `🔥 리뷰 급증`
- `📈 오늘 리뷰 +12`
- `👀 지금 많이 보는 상품`

**심리:** "남들이 많이 보는 건 나도 봐야지"

---

### 🏪 Section 3. "마트별 인기 상품" (차별화 포인트)

> CartPick의 핵심 차별점. 마트 단위 탐색.

**MVP 기본 노출:** GS25, CU (상위 2개)

**점진적 확장:** 세븐일레븐, 이마트24 → 스크롤/탭 확장

**쿼리 조건:**
- 해당 리테일러에서 판매 중 (`retailer_products.is_available = true`)
- 리뷰 평점 × 리뷰 수 기반 랭킹

**카드 구성:**
```
┌─────────────────┐
│   [상품 이미지]  │
├─────────────────┤
│ 곤약젤리        │
│ ⭐ 4.2 · 💬 47  │
│ 2,500원         │
└─────────────────┘
```

**유저 내면:** "지금 GS25 가는데 뭐 사지?" → 바로 답 제공

---

### 🎯 Section 4. 카테고리 탐색

> 가로 스크롤 아이콘 그리드. 탐색 진입점.

```
[🍪]   [🍬]   [🥤]   [🍜]   [🍦]   [🍱]
과자   젤리   음료   라면  빙과  간편식
```

**왜 중요:** 특정 카테고리 러버(젤리 덕후, 라면 덕후)의 깊은 탐색 시작점

---

### 💰 Section 5. "지금 할인 중"

> 후반부 배치지만 클릭률 높음.

**쿼리 조건:**
- `retailer_products.sale_price IS NOT NULL AND sale_price < price`

**뱃지:**
- `💰 GS25 1+1`
- `🏷️ CU 할인 중`
- 할인율 표시 (예: `-20%`)

**참고:** MVP 초기에는 행사 정보가 부족할 수 있음 → 데이터 부족 시 섹션 숨김 처리

---

## 4. UX 핵심 트리거

### 트리거 1: "지금" (시간성)

- "지금 핫한"
- "오늘 리뷰 +12"
- "이번 주 신상"

→ **시간 한정성 = 클릭 유도**

### 트리거 2: "마트" (현실 연결)

- "GS25 인기"
- "CU 추천"

→ **유저가 실제로 가는 편의점과 매칭**

### 트리거 3: "리뷰 상태" (행동 유도)

| 상태 | 표시 | 유도 행동 |
|------|------|-----------|
| 리뷰 0개 | 💬 첫 리뷰 작성 | 첫 리뷰어 뱃지 획득 |
| 리뷰 폭발 중 | 🔥 리뷰 급증 | 궁금증 → 클릭 |
| 평점 높음 | ⭐ 4.5 | 구매 전 확신 |
| 호불호 | 😵 호불호 갈림 | 논쟁 참여 유도 |

## 5. 홈 → 상품 상세 연결 플로우

```
홈 진입
  ↓
🔥 신상 섹션에서 카드 클릭
  ↓
상품 상세
  ↓
리뷰 0개 or 적음 발견
  ↓
"첫 리뷰 작성" CTA
  ↓
리뷰 작성 (3탭 이내)
  ↓
뱃지 획득 알림
```

이것이 CartPick의 **핵심 Activation 루프**.

---

## 6. 홈 카드 디자인 규칙

모든 상품 카드는 다음 요소를 포함한다:

```
┌─────────────────────┐
│                     │
│    [상품 이미지]     │  ← 정사각형
│                     │
│  [상태 뱃지]         │  ← 우상단 오버레이
├─────────────────────┤
│ 상품명 (2줄 max)    │
│ ⭐ 4.2 · 💬 47      │  ← 평점·리뷰 수
│ [리테일러명 or 가격] │  ← 컨텍스트
└─────────────────────┘
```

**상태 뱃지 종류:**
- `🆕 NEW` — 신상품
- `🔥 HOT` — 리뷰 급증
- `💬 첫 리뷰` — 리뷰 0개
- `💰 SALE` — 할인 중
- `😵 호불호` — 평점 분산 큰 상품

---

## 7. 절대 하면 안 되는 것

| ❌ 안티패턴 | 이유 |
|-------------|------|
| **섹션 7개 이상** | 스크롤 피로, 기억 불가. 5~6개가 최대 |
| **텍스트만 있는 리스트** | 시각 자극 부족. 반드시 카드형 |
| **"평점순"만 있는 정렬** | generic. 상황/시간/마트 축 없음 |
| **홈에 리뷰 힌트 없음** | 리뷰 앱 정체성 약화. 최소 리뷰 수/평점 노출 |
| **개인화 없는 "추천"** | MVP에서 "추천" 표현 금지. "인기"/"신상"/"할인" 등 명확한 이유 기반 |

---

## 8. 개인화 로드맵 (Post-MVP)

| Phase | 기능 |
|-------|------|
| **MVP** | 전역 인기/신상 (개인화 없음) |
| **Post-MVP v1** | 내가 자주 보는 카테고리 기반 섹션 추가 |
| **Post-MVP v2** | 즐겨찾기 기반 유사 상품 추천 |
| **Post-MVP v3** | 내가 리뷰한 상품 유사도 추천 |
| **Post-MVP v4** | 위치 기반 (근처 편의점) |

---

## 9. 섹션별 우선순위

### MVP 필수 (Must)
1. 🔥 이번 주 편의점 신상
2. 🏪 GS25 인기 상품
3. 🏪 CU 인기 상품
4. 🎯 카테고리

### MVP 권장 (Should)
5. 📈 지금 핫한 상품 (리뷰 데이터 충분 시)

### Post-MVP
6. 💰 지금 할인 중 (가격/행사 데이터 확보 후)
7. 🎁 개인화 추천

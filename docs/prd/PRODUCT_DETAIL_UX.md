# CartPick 상품 상세 페이지 UX 설계

## 0. 상품 상세의 목표

상품 상세 페이지의 역할은 단 하나:

> **"리뷰를 보게 만들고, 결국 쓰게 만든다"**

홈이 Discovery라면, 상품 상세는 **Conversion (리뷰 작성 전환)** 이 유일한 목표다.

## 2. 섹션별 상세 설계

### 🟢 Section 1. 상품 헤더 (첫인상)

```
┌─────────────────────┐
│                     │
│   [상품 이미지]      │
│                     │
├─────────────────────┤
│ 하리보 골드베렌 100g │
│ ⭐ 4.2  (47)         │
│ 🏪 GS25 · CU · 세븐  │
└─────────────────────┘
```

**핵심 포인트:**
- 상품 이미지는 **정사각형** (1:1)
- 평점 옆에는 **리뷰 수 괄호 표시** (리뷰 앱 정체성)
- 리뷰 0개면 별점 대신 `💬 첫 리뷰 대기 중` 표시
- **판매 리테일러는 로고로 즉시 노출** → "이거 어디서 사?" 질문에 즉답

**데이터 바인딩:**
- `product.image_url`
- `product.name`
- `product.average_rating` / `product.review_count`
- `product.retailers[]` (→ 로고 리스트, 최대 4개, 넘으면 `+N`)

---

### 🟡 Section 2. 한 줄 요약 (엄청 중요)

```
┌─────────────────────────────────────┐
│ 💬 "달긴 한데 계속 손이 가는 젤리"   │
└─────────────────────────────────────┘
```

**왜 중요한가:**
- 유저는 긴 리뷰를 안 읽는다
- **한 줄 판단 → 살까/말까 결정**
- 리뷰 앱의 "핵심 가치" 응축

**구현 단계:**

| Phase | 방법 |
|-------|------|
| **MVP** | 관리자 수동 입력 (상품 등록 시 `summary_line` 필드) |
| **Post-MVP v1** | 가장 좋아요 많은 리뷰의 title을 자동 발췌 |
| **Post-MVP v2** | LLM 기반 리뷰 요약 (Claude API 등) |

**리뷰 0개일 때:** 이 섹션 자체를 숨김 (빈 인용구는 신뢰 저하)

---

### 🔵 Section 3. 상단 리뷰 작성 CTA (반드시 상단에)

```
┌─────────────────────────────────────┐
│    [ ✍️  리뷰 쓰기 ]                │
└─────────────────────────────────────┘
```

**왜 상단인가:**
- **이미 먹어본 사람**은 리뷰 읽을 필요 없이 바로 쓰고 싶다
- 스크롤 해야 작성 버튼 찾으면 이탈
- 하단 고정 CTA와 **중복이어도 OK** — 작성 의욕 최고점을 놓치지 말 것

**버튼 상태 분기:**

| 상태 | 표시 |
|------|------|
| 내가 아직 리뷰 안 씀 | `✍️ 리뷰 쓰기` |
| 내가 이미 리뷰 씀 | `📝 내 리뷰 수정하기` |
| 로그인 안 됨 | `✍️ 리뷰 쓰기` (클릭 시 로그인 플로우) |

---

### 🟣 Section 4. 구매 가능 마트

```
┌─────────────────────────────────────┐
│ 📍 구매 가능한 곳                    │
│                                     │
│ ┌─────────┐ ┌─────────┐             │
│ │ GS25    │ │ CU      │             │
│ │ 2,500원 │ │ 2,800원 │             │
│ └─────────┘ └─────────┘             │
│ ┌──────────┐                         │
│ │ 세븐일레븐 │                        │
│ │ 2,500원   │                        │
│ └──────────┘                         │
└─────────────────────────────────────┘
```

**기능:**
- 리테일러 카드 클릭 시 → **그 리테일러에서 구매한 리뷰만 필터링**
- 할인 중이면 `💰 SALE` 뱃지 표시
- 가격 정보는 참고값 (`last_synced_at` 기준 "N일 전" 표시)

**제약:**
- 품절(`is_available = false`)은 흐리게 처리
- `launch_status = 'coming_soon'` 리테일러는 노출하지 않음

---

### 🟠 Section 5. 리뷰 요약 (키워드 기반)

```
┌─────────────────────────────────────┐
│ 👍 많이 언급된 키워드                │
│                                     │
│ [달다 32]  [쫀득 24]  [양 적음 12]  │
│ [재구매 O 78%]  [호불호 없음]       │
└─────────────────────────────────────┘
```

**왜 중요한가:**
- 유저는 빠른 판단을 원함
- 키워드 = 리뷰 읽기 없이도 감 잡기

**MVP 구현:**
- **태그 기반**: 경량 리뷰 작성 폼의 체크박스 태그 집계
  - "맛있어요" N명, "가성비 좋아요" N명, "양이 적어요" N명
- **재구매율**: `reviews.repurchase_intent = true` 비율

**Post-MVP 구현:**
- 리뷰 본문에서 형태소 분석 → 자주 등장한 명사/형용사 추출

---

### 🔴 Section 6. 리뷰 리스트 (핵심 콘텐츠)

```
┌─────────────────────────────────────┐
│ 리뷰 47개                            │
│                                     │
│ [전체] [GS25] [CU] [세븐]           │
│ [최신순 ▾]                          │
├─────────────────────────────────────┤
│ ┌─────────────────────────────────┐│
│ │ 🥇 젤리러버  GS25에서 · 3일 전  ││
│ │ ⭐⭐⭐⭐☆                         ││
│ │                                 ││
│ │ "생각보다 맛있음"                ││
│ │ 달아서 호불호 있을 것 같긴       ││
│ │ 한데 저는 맛있게 먹었어요...     ││
│ │                                 ││
│ │ 👍 12      💬 답글 2            ││
│ └─────────────────────────────────┘│
│                                     │
│ ┌─────────────────────────────────┐│
│ │ (다음 리뷰)                      ││
│ └─────────────────────────────────┘│
└─────────────────────────────────────┘
```

**필터:**
- 전체 / 리테일러별 (GS25, CU, 세븐일레븐, 이마트24)
- 정렬: 최신순 / 도움순(좋아요) / 평점 높은 / 평점 낮은

**카드 규칙:**
- 리뷰 본문은 **3줄 이상 말줄임** → 탭하면 전체
- 첫 리뷰어 뱃지(`🥇`) 아이콘 표시
- 구매처 리테일러 배지 표시
- 긴 리뷰 카드 ❌, 짧고 빠르게 스캔 가능한 카드 ⭕

---

### 하단 고정 CTA (Sticky Bottom Bar)

```
┌─────────────────────────────────────┐
│      [ ✍️  리뷰 작성하기 ]          │  ← 스크롤해도 항상 보임
└─────────────────────────────────────┘
```

**왜 하단 고정인가:**
- 유저 행동 패턴: "리뷰 읽다가 → 나도 쓰고 싶어짐"
- **이 타이밍을 놓치지 말 것**
- 스크롤 위치 무관하게 즉시 작성 진입 가능

**상단 CTA와의 역할 분담:**
- 상단: 이미 먹어본 유저의 즉시 작성
- 하단: 리뷰 읽다가 전환된 유저의 즉시 작성

## 4. 리뷰 작성 진입 UX (초간단)

CTA 클릭 시 **바텀시트** 또는 **전체 화면**으로 즉시 작성 폼:

```
┌─────────────────────────────────────┐
│  × 닫기                              │
│                                     │
│  하리보 골드베렌 100g                 │
│                                     │
│  별점                                │
│  ⭐️⭐️⭐️⭐️☆                        │
│                                     │
│  어디서 구매하셨나요? (선택)          │
│  [GS25 ✓] [CU] [세븐] [이마트24]   │
│                                     │
│  한 마디로 (태그 다중 선택)           │
│  [ ] 맛있어요                        │
│  [✓] 가성비 좋아요                   │
│  [ ] 양이 적어요                     │
│  [✓] 재구매 의사 있음                │
│                                     │
│  한 줄 평 (선택)                     │
│  ┌─────────────────────────────┐    │
│  │                             │    │
│  └─────────────────────────────┘    │
│                                     │
│      [ 리뷰 등록 ]                   │
└─────────────────────────────────────┘
```

**원칙:**
- **별점만 있으면 제출 가능** — 필수 입력 최소화
- 태그 체크박스로 빠른 입력
- 한 줄 평은 선택 사항
- **제출까지 10초 이내**

## 5. UX 핵심 트리거 메시지

| 트리거 | 메시지 | 사용처 |
|--------|--------|--------|
| **첫 리뷰** | "첫 리뷰어가 되어보세요" | 리뷰 0개 |
| **도움됨** | "다른 사람에게 큰 도움이 됩니다" | 리뷰 0~3개 |
| **간단함** | "10초면 충분해요" | CTA 하위 텍스트 |
| **희소성** | "아직 리뷰가 부족해요" | 리뷰 1~3개 |
| **사회적 증명** | "N명이 도움 받았어요" | 리뷰 많음 |

---

## 6. 절대 하면 안 되는 것

| ❌ 안티패턴 | 이유 |
|-------------|------|
| **리뷰 작성 깊은 단계** | 가입 → 작성 → 확인 → 제출 → 90% 이탈 |
| **텍스트만 많은 화면** | 요약(한 줄평, 키워드) 없으면 스캔 불가 |
| **CTA 없는 구조** | 리뷰가 쌓이지 않음 |
| **상단 CTA 누락** | 이미 먹어본 유저의 즉시 작성 기회 상실 |
| **하단 고정 CTA 없음** | 리뷰 읽다가 작성 전환 타이밍 놓침 |
| **긴 리뷰 카드 기본 펼침** | 스캔 불가, 이탈 증가 |
| **리뷰 0개 UI가 그냥 "리뷰 없음"** | Activation 기회 낭비 |


## 8. 데이터 요구사항 (ERD 영향)

상품 상세 페이지를 위해 필요한 데이터:

| 필드 | 위치 | 비고 |
|------|------|------|
| `products.summary_line` | 신규 추가 | 한 줄 요약 (MVP는 관리자 수동 입력) |
| `products.rating_distribution` | 계산값 | 별점별 개수 (쿼리 또는 비정규화) |
| `products.tag_counts` | 계산값 | 태그별 언급 수 (리뷰 집계) |
| `products.repurchase_rate` | 계산값 | `repurchase_intent` 비율 |
| `reviews.has_first_reviewer_badge` | 조인 | 유저가 이 상품의 첫 리뷰어인지 |

## 9. API 영향

### `GET /api/v1/products/:productId` 응답 확장

```json
{
  "product": {
    "id": 101,
    "name": "하리보 골드베렌 100g",
    "summary_line": "달긴 한데 계속 손이 가는 젤리",
    "image_url": "...",
    "brand": {...},
    "category": {...},
    "average_rating": 4.2,
    "review_count": 47,
    "rating_distribution": {
      "5": 18, "4": 15, "3": 8, "2": 4, "1": 2
    },
    "tag_counts": {
      "taste_good": 32,
      "value_good": 24,
      "amount_small": 12
    },
    "repurchase_rate": 78.5,
    "retailers": [...],
    "my_review": {
      "exists": false,
      "review_id": null
    }
  }
}
```

**추가 필드:**
- `summary_line`: 한 줄 요약
- `tag_counts`: 태그별 집계
- `my_review`: 현재 유저의 리뷰 존재 여부 (상단 CTA 분기용)
